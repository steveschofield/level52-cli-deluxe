### 4.1.3 XSLT Injection leading to Server-Side Request Forgery (SSRF) Severity: CRITICAL Affected Component: `instructions.php` CVSS v3.1: 9.8 (AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H) OWASP Top 10: A03:2021 - Injection CWE: CWE-611 (XSLT Injection), CWE-918 (Server-Side Request Forgery)

Technical Description: The application parameter `doc` in `instructions.php` is vulnerable to XSLT Injection. An attacker can supply a malicious XSLT payload that forces the server to fetch and process external resources, including internal network resources. This was confirmed by successfully retrieving data from an internal SSH service (port 22).

Evidence & Proof of Concept: The scanner successfully executed the following payload: `http://192.168.1.49:4280/instructions.php?doc=%3Cxsl%3Avalue-of+select%3D%22document%28%27http%3A%2F%2F192.168.1.49%3A22%27%29%22%2F%3E` Response: `200|40585|0.002319` (Indicates successful retrieval of the target resource).

Impact Analysis: This vulnerability allows an attacker to: Access internal services (e.g., SSH, metadata APIs, admin panels) not exposed to the public internet. Perform blind SSRF to scan the internal network. Potentially exfiltrate sensitive data or execute code on the server.

Exploitation Information: CISA KEV Status: N/A CVE Identifiers: N/A Metasploit Modules: N/A Exploit-DB IDs: N/A GitHub PoCs: N/A Exploitation Attempt: Successful. The scanner confirmed the ability to fetch internal resources via the `document()` function in XSLT.

Remediation Plan: 1. Input Validation: Strictly validate and sanitize all user input passed to XSLT processors. 2. Disable External Access: Configure the XSLT processor to disable the `document()` function and external entity resolution. 3. Network Controls: Implement egress filtering to prevent the web server from initiating connections to internal network segments. 4. Code Review: Audit the `instructions.php` file and remove or secure the XSLT processing logic.

Impact Analysis: Without CSP, the browser is more permissive in loading resources, making the application more susceptible to XSS attacks where malicious scripts are injected and executed in the victim's browser.

Exploitation Information: CISA KEV Status: N/A CVE Identifiers: N/A Metasploit Modules: N/A Exploit-DB IDs: N/A GitHub PoCs: N/A Exploitation Attempt: N/A

Remediation Plan: 1. Implement CSP: Configure the web server to send a strict CSP header. Example: `Content-Security-Policy: default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline';` 2. Audit: Review the application to ensure legitimate scripts and styles are allowed by the policy.
