### Technical Description The application fails to enforce authentication controls on the order creation endpoint. The API accepts `POST` requests to `/orders` without validating the presence or validity of an `Authorization` header. This allows any unauthenticated actor to create orders within the system, leading to unauthorized data modification and potential business logic abuse.

### Evidence and Proof of Concept Automated testing via `schemathesis` confirmed that the endpoint returns a `200 OK` status code when the `Authorization` header is missing or invalid, whereas a `401 Unauthorized` response was expected.

Test Case 1: Request sent without `Authorization` header. Expected: `401 Unauthorized` Actual: `200 OK` Test Case 2: Request sent with invalid/generated auth token. Expected: `401 Unauthorized` Actual: `200 OK`

Business Impact: Potential for financial loss, inventory depletion, and data integrity corruption.
